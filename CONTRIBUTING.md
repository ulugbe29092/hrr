# Contributing to Do'kon Tizimi

Loyihaga hissa qo'shganingiz uchun rahmat! 🎉

## 🚀 Qanday hissa qo'shish mumkin?

### 1. Fork qiling

Loyihani o'z GitHub akkauntingizga fork qiling.

### 2. Clone qiling

```bash
git clone https://github.com/your-username/dokon-tizimi.git
cd dokon-tizimi
```

### 3. Branch yarating

```bash
git checkout -b feature/yangi-funksiya
```

### 4. O'zgarishlar kiriting

- Kod yozing
- Test qiling
- Commit qiling

```bash
git add .
git commit -m "feat: yangi funksiya qo'shildi"
```

### 5. Push qiling

```bash
git push origin feature/yangi-funksiya
```

### 6. Pull Request yarating

GitHub da Pull Request yarating va o'zgarishlarni tushuntiring.

## 📝 Kod Standartlari

### TypeScript

- Strict mode ishlatiladi
- Type safety muhim
- `any` dan qochish kerak

### Komponentlar

- Functional components ishlatiladi
- React Hooks ishlatiladi
- Props uchun interface yaratiladi

### Naming Conventions

- Components: PascalCase (`Button.tsx`)
- Functions: camelCase (`getUserData`)
- Constants: UPPER_SNAKE_CASE (`API_URL`)
- Files: kebab-case (`user-profile.tsx`)

### Commit Messages

Conventional Commits formatidan foydalaning:

- `feat:` - Yangi funksiya
- `fix:` - Bug fix
- `docs:` - Dokumentatsiya
- `style:` - Kod formatlash
- `refactor:` - Kod refaktoring
- `test:` - Test qo'shish
- `chore:` - Boshqa o'zgarishlar

Misol:
```
feat: mahsulot qo'shish funksiyasi qo'shildi
fix: login sahifasidagi xato tuzatildi
docs: README yangilandi
```

## 🧪 Testing

Yangi funksiya qo'shishdan oldin test yozing:

```bash
npm run test
```

## 📋 Pull Request Checklist

- [ ] Kod ishlaydi va test qilingan
- [ ] Linting xatolari yo'q (`npm run lint`)
- [ ] Commit messages to'g'ri formatda
- [ ] Dokumentatsiya yangilangan (kerak bo'lsa)
- [ ] TypeScript xatolari yo'q
- [ ] Responsive dizayn tekshirilgan

## 🐛 Bug Report

Bug topilsa, GitHub Issues da xabar bering:

1. Bug tavsifi
2. Qayta ishlab chiqarish qadamlari
3. Kutilgan natija
4. Haqiqiy natija
5. Screenshot (agar kerak bo'lsa)

## 💡 Feature Request

Yangi funksiya taklif qilish uchun:

1. Funksiya tavsifi
2. Nima uchun kerak?
3. Qanday ishlashi kerak?
4. Mockup yoki misol (agar bor bo'lsa)

## 📞 Aloqa

Savollar bo'lsa, GitHub Issues da so'rang.
